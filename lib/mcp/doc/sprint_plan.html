Правила работы со спринтами
Перед спринтом
  • Прочитать раздел «Подготовка» полностью
  • Убедиться, что все зависимости спринта выполнены
  • Запросить спринт одним сообщением согласно шаблону запроса
  • Не начинать следующий спринт пока текущий не зачекирован
Во время спринта
  • Один спринт = один запрос к AI (не дробить)
  • Все файлы пишутся сразу с тестами
  • Код проверяется командой dart test в конце
  • Сохранять структуру монорепо строго по плану
После спринта
  • Запустить все команды из раздела «Проверка»
  • Убедиться что все тесты зелёные
  • Зафиксировать git commit с тегом спринта
  • Только после этого — переход к следующему

Sprint 2 · Протокол
aq_mcp_core — Protocol Layer
Полная реализация MCP протокола: JSON-RPC codec, STDIO transport с Content-Length framing, HTTP transport, MCP lifecycle (initialize handshake). Никакой бизнес-логики — только protocol.
aq_mcp_core depends: aq_schema shelf
~4ч
Расчётное время
Критический путь
Подготовка
Sprint 0 и Sprint 1 завершены. Все тесты зелёные. melos bootstrap выполнен успешно. Понимание MCP spec: JSON-RPC 2.0 over STDIO.
Задачи спринта
  • McpCodec — decode(String raw) → McpRequest, encode(McpResponse) → String, encodeError(McpError) → String. Обработка всех JSON-RPC edge cases.
  • StdioTransport — строгий Content-Length framing: Content-Length: N\r\n\r\n{json}. Stream<McpRequest> на входе, send() на выходе. stdout = только JSON, stderr = логи.
  • HttpTransport — shelf-based, POST /mcp → JSON-RPC, GET /health → 200 OK. Keep-alive соединения.
  • McpLifecycle — handleInitialize(request) → McpInitializeResponse с capabilities + _aq_extensions. Флаг initialized. Reject любых запросов до initialize.
  • Error mapping — таблица кодов: -32700 parse, -32600 invalid, -32601 not found, -32602 params, -32603 internal, -32000..-32004 AQ extensions
  • McpRequestRouter — базовый router: dispatch по method, возвращает Future<McpResponse>. Без конкретных handlers (они в adapter).
  • Тесты — codec round-trip (encode→decode→encode == исходный), STDIO framing корректность, initialize handshake
Критически важно
STDIO framing — строго соблюдать: каждый JSON-RPC message предваряется заголовком Content-Length: N\r\n\r\n. Никаких partial writes. Никакого non-JSON в stdout.
McpLifecycle — любой запрос до initialize должен возвращать ошибку -32600 (Invalid Request), кроме самого initialize.

  • Schema validation — перед вызовом execute(): validate payload против inputSchema инструмента
  • Timeout wrap — meta.timeout_ms → Future.timeout. При timeout → WorkerResult(status: timeout)
  • Error normalization — любой exception → WorkerResult(status: failed, error: normalized)
  • Health loop — Timer.periodic, каждые interval секунд: POST /workers/health
  • Graceful shutdown — ProcessSignal.sigterm.watch() → завершить текущие jobs → отправить unhealthy → exit
  • Пример воркера — workers/dart_example: tool echo + tool uppercase. Dockerfile. README.
